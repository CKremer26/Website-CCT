Publishing and Heroku


Heroku is a cloud based application platform. Basically, it is a website that allows us to publish our websites to be hosted on their servers. There are other platforms that do this, but in this course we use Heroku to publish our web pages, infact, as you probably already guessed, this website is also being hosted on Heroku. Of course, if you want your website to be public and used for something like a business, you're going to have to purchase a domain from a service like GoDaddy.

How to Publish Using Heroku

Typically, we would take our code from Glitch and upload it to Github, and from there, put it onto Heroku. However, there were issues involving a security breach with Heroku at the time so this did not work. As an alternative, we used the Glitch terminal to port it directly to Heroku.

  • Create a Heroku account and click the "new" button and hit create new app. Name it whatever you want and click create app.
  • You are then going to want to navigate to your account settings and look for your api key. Copy this key for later.
  • Go back to your dashboard and be sure that you are in the deploy tab of your project.
  • Now, go back to your Glitch project and in the bottom bar, click terminal. Now put in the command "npm install heroku".
  • After it is installed, use the command "heroku login -i". Your password is your api key.
  • Follow the instructions on Heroku, negating the cd command. Once you do that, your page should be live!
