GitHub


GitHub is a cloud based service that allows us to upload our gits to the cloud. This is very useful when sharing, collaborating, and transfering projects. You can make your repository public so others can download your source code. You can also add collaborators to your project repository so they can pull the code to their local IDE, make changes, and either make a new git branch or request merges to the master branch. And more importantly, especially in this course, it allows the transfer of source code from one machine to the other.

To extend off that last sentence, I often used GitHub to take my projects from the computer at the school, and put them on my laptop at home through GitHub. However, I did not do this when I was developing my Flutter app which proved to be a big mistake. One day when I came into class, the computer I was using could find a boot device, and because of the password protection, we couldn't get the computer to boot up Windows. Because all my Flutter projects were on that computer's hard drive, I could not access my Flutter projects. What we ended up doing was taking the hard drive out of the computer and transfering the files to a different working computer. This is why it is SO IMPORTANT to upload your projects to GitHub, because had I done that, I could've just pulled the repository to my IDE and have all my work saved, without having to physically transfer the files.

How to Use it

In order to put our local git repository on GitHub, you must first make a GitHub account. After you do that, follow these steps:

  • Create a new repository in GitHub by clicking new. Name it whatever you want and click create.
  • In your repository, you will see a hyperlink. Copy this for later.
  • Now, back to your IDE and assuming you already initialized a repository, you will have to connect that repo to your GitHub repo. To do this, type in the command $ git remote add origin (Link copied from earlier)
  • Input the command $ git remote show origin to check if your repo is connected.
  • Now, you can push your master branch onto your GitHub repo by using the command $ git push origin master. You should now be able to see your code in your GitHub repo. If you want to push other branches, just put the branch name instead of master.
  • It is also important that you pull code from your GitHub repo if someone else is working on it or you made a change in the Github editor. To do this just input the command $ git pull origin (branch name).

GitHub Gists

GitHub also provides the service of gists, which is basically snippets of code. With GitHub, you can actually imbed gists into a webpage, which I have done with my code from Caesar and Mario. You can do this by going to GitHub Gists. Click create gist and when you are done, copy the script tag in the upper right corner. Just put this script into your source code and it should show up. You can edit the gist from GitHub and it will update automatically on your website.
