What is Flutter?


Flutter is an open-sourced UI software development kit created by Google. It is used to develop apps cross platform on Android, iOS, Linux, macOS, and Windows. It uses the programing language Dart. In this course, we learned how to use Flutter to create basic apps as it is somewhat similar to C.

How to Install Flutter

The IDE we used was VS Code, so make sure you have VS Code installed on your machine.

  • First, install the Flutter SDK for Windows, this may take a while to install.
  • Create a folder in your Windows C drive called src. Extract the contents of the Flutter SDK into that folder.
  • In the Windows search bar, search "env" and click on Edit the system environment variables. A window will open, click environment variables. Under the user variables, click edit and create new path. Set the new path as C:\src\flutter\bin. Save the changes.
  • Open up VS Code and navigate to the extensions tab on the left. Search for Flutter and install the first extension you see.
  • Go back to view and open the command palette and search flutter. Click new project, application. You can save your project in a new folder in your doccuments folder. Name the app anything you want (preferably camel cased).
  • The new project made will be of a template app. main.dart is where you will be coding. At the bottom right will be a button saying Windows, switch this to chrome or to the emulator you are using.
  • Now press F5 and your code will run and open the emulator/chrome tab!

How to Use Flutter

Flutter uses things called "widgets"to build apps. These widgets are built as a tree, known as a widget tree. You can give these widgets certain properties, children, and attributes. The difference between childs and children is a child means it is just one child of the widget, ex: containers only use childs. Children refers to the widget having multiple widgets as a child, usually in square brackets (like an array) ex: columns and rows use children. Here is a list of the most commonly used widgets/classes that we used:

  • Scaffold - A widget that apply the basic material design layout. It stretches to fill the entire screen. Contains the body property.
  • Column - A widget that stacks its children on top of each other.
  • Row - A widget that places its children side by side next to each other.
  • Container - A widget that can have only one child.
  • Center - A widget that centers its child to the center of the screen.
  • SafeArea - A widget that insets its child to avoid areas where it might be obstructed (bezels of a phone).
  • Text - A widget that displays text
  • MaterialApp - A widget that wraps a number of widgets that are commonly required for material design application. Usually the first widget of the main page. (return MaterialApp())

There are also many properties of the widgets that can be changed, such as axis alignment, color, and margins. Tutorials on how to do this can be found here. Note that there are properties that only certain widgets have.
