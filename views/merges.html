Merges and Merge Conflicts


A git merge conflict is where there is a failure to merge a branch to the master branch. There are many reasons why this can happen and is probably one of the most common errors to encounter with Git, espicially when collaborating with others. A merge error most commonly occurs when one person creates a branch, changes something, and commits, while another person is on the master branch, changes the same thing, and commits. Because the code is conflicting each other, the git merge command will merge what it can, and it is up to the developer to resolve the conflict. Another type of error is version mismatch, where someone is trying to merge a branch of a previous version of master with a newer version of master.

How to Resolve Merge Conflicts

It is important to note that merge conflicts have to be resolved manually, usually by the leader if a group of people are collaborating together. The first step in solving a merge conflict is to find what file is conflicting and what is conflicting in the file in question. Using the $ git status command, we can see an output of unmerged paths, these are the paths that need to be resolved. The best way to resolve a merge conflict is to edit the conflicted file directly, and make appropiate changes that resolve the conflict. Add the file to the staging are and commit it. Git will see that the conflict is resolved and creates a new commit to finialize the merge. A good tutorial for recognizing and solving merge conflicts

Other Useful Commands to Solve Merge Conflicts

  • $git log --merge - This command will show you a log with a list of commits that conflict between merging branches.
  • $git diff - This command will show you differences between states of repositories/files which is useful when avoiding and predicting merge conflicts.
  • $git reset --mixed - This command will undo changes to the working directory and staging area, useful if you make a mistake.
  • $git merge --abort - This command stop the merge and return the branch to a state before the merge took place.
  • $git reset - This command can be used during a merge conflict and will return the conflicted files to a state where they were not conflicting.

An important thing to note is: always protect the master branch. This is where all your code is and you don't want to lose it. Changes should only ever be merged via pull requests, as good practice, that way every merge will need to be reviewed by the leader as to prevent loss of work and merge conflicts.
